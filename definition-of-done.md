# Definition of Done

[cite_start]A backlog item is Done when ALL of the following criteria are met[cite: 3].

### Code Quality
* [cite_start]Code committed to the main branch[cite: 5].
* [cite_start]Application starts without errors[cite: 6].
* [cite_start]Code follows existing project patterns[cite: 7].

### Testing
* [cite_start]At least one unit test covers the core logic[cite: 9].
* [cite_start]At least one integration test covers the HTTP endpoint[cite: 10].
* [cite_start]All existing tests continue to pass (no regressions)[cite: 11].
* [cite_start]Test coverage for new code is at least 80%[cite: 12].

### CI/CD
* [cite_start]CI pipeline passes on the feature commit[cite: 14].
* [cite_start]Tests run automatically via GitHub Actions[cite: 15].
* [cite_start]No pipeline failures left unresolved[cite: 15].

### Acceptance Criteria
* [cite_start]All acceptance criteria in the backlog item are verified[cite: 17].
* [cite_start]Feature works correctly for both happy-path and error cases[cite: 18].

### Documentation
* [cite_start]README updated if a new endpoint or behavior is added[cite: 20].
* [cite_start]Sprint review document references the completed item[cite: 21].
